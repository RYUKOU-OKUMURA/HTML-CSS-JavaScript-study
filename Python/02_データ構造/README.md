## 📊 データ構造の比較

| データ構造 | 順序 | 変更可能 | 重複 | 主な用途 |
|----------|------|---------|------|---------|
| リスト (list) | あり | 可 | 可 | 順序が重要なデータの集合 |
| タプル (tuple) | あり | 不可 | 可 | 変更されない固定データ |
| 辞書 (dict) | あり※ | 可 | キー:不可<br>値:可 | キーと値のペア |
| セット (set) | なし | 可 | 不可 | 重複のないデータの集合 |

※Python 3.7以降、辞書は挿入順序を保持します

## 💡 データ構造の選び方

### リストを使う場合
- 順序が重要
- 同じ値が複数回出現する可能性がある
- 要素の追加・削除が頻繁

```python
shopping_list = ["りんご", "バナナ", "りんご"]  # 重複OK
scores = [85, 92, 78, 95, 88]  # 順序が重要
```

### 辞書を使う場合
- キーと値のペアで管理したい
- キーで高速に値を検索したい

```python
user = {"name": "太郎", "age": 25, "city": "東京"}
prices = {"apple": 100, "banana": 80, "orange": 120}
```

### タプルを使う場合
- データを変更されたくない
- 複数の値を返す関数

```python
coordinates = (35.6762, 139.6503)  # 座標（変更不可）
rgb = (255, 128, 0)  # RGB値（変更不可）
```

### セットを使う場合
- 重複を自動的に除去したい
- 集合演算（和集合、積集合など）を行いたい
- 要素の存在確認を高速に行いたい

```python
unique_tags = {"python", "django", "flask", "python"}  # 重複が削除される
```

## 🎨 Pythonic なデータ操作

### アンパッキング（展開）

```python
# リストのアンパッキング
x, y, z = [1, 2, 3]

# 残りをまとめて取得
first, *rest = [1, 2, 3, 4, 5]
print(first)  # 1
print(rest)   # [2, 3, 4, 5]

# 辞書のアンパッキング
user = {"name": "Alice", "age": 25}
updated_user = {**user, "city": "Tokyo"}
# {'name': 'Alice', 'age': 25, 'city': 'Tokyo'}
```

### enumerate() と zip()

```python
# インデックスと値を同時に取得
fruits = ["apple", "banana", "cherry"]
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")

# 複数のリストを並行してループ
names = ["Alice", "Bob"]
ages = [25, 30]
for name, age in zip(names, ages):
    print(f"{name}: {age}歳")
```
